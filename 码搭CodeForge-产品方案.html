企业级 AI 前端页面工厂：理解业务需求，复用企业组件，自动生成可运行页面

码搭不是又一个代码补全工具。它是一个懂公司组件库、懂业务规范、懂页面模式的 AI 工程师。 输入需求/原型/接口文档，直接输出基于公司技术栈的可运行前端页面——不是代码片段，不是骨架，是完整的、符合规范的、可以跑起来的页面。

🤖 Agent 编排方案

🎯

Intent Agent · 意图解析

接收用户输入，识别页面类型（列表/表单/详情/混合），提取字段需求和交互需求，输出结构化的页面描述JSON。

🔍

Retrieval Agent · 组件检索

基于页面描述，在RAG知识库中检索最匹配的企业组件，输出组件清单+使用方式+Props配置。

⚡

CodeGen Agent · 代码生成

融合页面描述、组件清单、编码规范，生成完整的Vue/React页面代码。含模板、逻辑、样式、类型定义。

Prompt 工程核心策略：不直接让LLM输出代码，而是采用「意图解析 → 结构设计 → 组件匹配 → 代码生成」的分步策略。每一步都有明确的输入输出约束，大幅提升生成质量。结构设计阶段使用Few-shot示例注入组件使用模式，代码生成阶段严格约束只使用企业组件。

10 项目创新亮点
与现有AI编程工具的差异化创新
🏭

创新1：从「代码助手」到「页面工厂」

不是生成代码片段，而是生成完整的、可运行的页面。包含组件编排、数据流、接口调用、Mock数据——拿来就能跑。

📚

创新2：企业组件库深度认知

通过RAG将公司组件库文档、示例、规范向量化，AI生成代码时100%使用企业组件，天然符合团队规范。这是Cursor/Copilot做不到的。

🔗

创新3：需求→接口→页面一体化

打通「PRD → 页面结构 → 组件匹配 → API对接 → Mock数据」全链路。不是单点AI增强，而是流程级自动化。

💬

创新4：页面级对话记忆

不只是聊天上下文，而是页面级的完整状态记忆。用户说「把表格改成卡片」，AI知道当前页面的所有组件、数据、状态，精准修改而不破坏其他部分。

🧩

创新5：Agent多步骤编排

不依赖单次LLM调用生成代码，而是通过多个专业化Agent（意图解析→组件检索→代码生成）协作完成，每个Agent专注自己的领域，质量更高、可控性更强。

🔄

创新6：页面模板沉淀与复用

每次生成的页面自动沉淀为可搜索的模板，团队积累越多，AI越「懂」业务。从单次生成升级为组织知识沉淀，形成正向飞轮。

  • SaaS + 私有化部署双模式
  • 按团队规模/生成次数定价
